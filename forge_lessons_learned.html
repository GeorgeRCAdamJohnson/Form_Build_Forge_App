Lessons Learned: Sci-Fi Form Builder & Atlassian Forge Migration

Executive Summary

Migrating a React app to Atlassian Forge—especially using UI Kit 1 or 2—proved far more difficult and limiting than anticipated. Forge’s security-first, sandboxed environment fundamentally restricts creative freedom, visual differentiation, and even standard React/HTML development patterns. Despite a full migration to UI Kit 2, many frustrations remained, leading to the decision to pursue a Custom UI approach for v3.

Key Lessons & Pain Points

  1. Migrating React to Forge is Hard and Risky
    • Direct migration is not possible: Forge UI Kits are not standard React; they lack HTML elements, DOM access, and custom CSS.
    • UI Kit 1 is deprecated: Using it leads to cryptic errors (“App update required”) and wasted debugging time.
    • UI Kit 2 is only a partial solution: Still highly restrictive, with limited components and no real styling freedom.
    • Custom UI is the only way to use real React, but comes with its own complexity (static builds, resource mapping, manifest management).
  2. Forge is Difficult and Restrictive
    • No custom CSS, no HTML elements, no animations: All apps look the same (“design token prison”).
    • Sandboxed execution: No direct API calls from frontend, no external libraries, no DOM manipulation.
    • Complex architecture: Requires resolvers for all backend logic, even for simple storage or API calls.
    • Misleading errors and warnings: Deprecation and build warnings are critical—never ignore them.
  3. UI Kit 1 & 2 Migration Wastes Time
    • Migration is a rewrite, not a refactor: Syntax, imports, and even component logic must change.
    • Mixed codebases are dangerous: Having both UI Kit 1 and 2 in the same repo causes confusion and bugs.
    • File/folder naming matters: Inconsistent organization slows debugging and migration.
  4. Forge is Not for Creative or Visually Distinct Apps
    • No visual differentiation: All Forge apps look corporate and boring.
    • No support for custom themes, branding, or advanced UI/UX.
    • Browser extensions or standalone React apps are better for creative projects.

What Actually Works in Forge

  • Standard React hooks and state management (in UI Kit 2 and Custom UI).
  • Basic XCSS styling (padding, margin, borderRadius).
  • API integration via resolvers.
  • Storage persistence through Forge APIs.
  • Templates and standardized patterns for admin/enterprise use cases.

Best Practices for Future Projects

  1. Treat warnings as errors: Never ignore deprecation or build warnings.
  2. Automate checks: Add scripts to block UI Kit 1 imports or deprecated patterns.
  3. Document everything: Clearly state which UI Kit or approach each file uses.
  4. Standardize structure: Use clear, consistent folder and file naming.
  5. Plan for migration: Expect that Atlassian will deprecate features and require rewrites.
  6. Use Custom UI for real React: Only Custom UI allows standard React, HTML, and CSS (with more setup).
  7. Consider alternatives for creative needs: Browser extensions or standalone apps offer true creative freedom.

Strategic Recommendations

  • Use Forge only for secure, functional, admin-focused apps.
  • For creative, visually distinct, or highly interactive apps, use Custom UI, browser extensions, or standalone React.
  • Maintain a knowledge base of migration scripts, code snippets, and lessons learned.
  • Develop and use templates for common Forge app patterns.

What the New Project Can Learn

  • Start with Custom UI if you want real React and CSS.
  • Keep the codebase clean and modular—separate UI, backend, and knowledge base logic.
  • Expect to spend significant time on manifest/resource mapping and static build output for Custom UI.
  • Document all workarounds, limitations, and migration steps for future reference.
  • Don’t chase the “sci-fi theme” dream in Forge UI Kit—focus on functionality and use Custom UI for any real theming.

Final Thoughts

Forge is a powerful but highly restrictive platform. It is best suited for secure, functional, and visually uniform apps. For anything creative, innovative, or visually distinct, Forge will fight you every step of the way. Custom UI is the only viable path for modern React development within Forge, but even then, expect extra setup and ongoing limitations.

The dream of sci-fi themed Jira interfaces lives on—but not within Forge’s walled garden.
